Getting started

  • In case of error messages, someone might have seen it too, so you can try a Google search for a quick fix.
 

Starting the app

Since this is ReactJS based application you need a server (Apache, IIS, xampp, etc) to serve the html files and perform http request to load all views.
 
Important! Opening the index.html with a double click (i.e. using file:// protocol) will show you only a blank page because there’s no server that response to the requests made for each view in order to display the app interface.

Integrated server (BrowserSync)

See section Build to install node and bower dependecies and then run gulp task to automatically serve the app

Alternative server setup (requires nodejs)

This is simple solution for a basic server setup using nodejs that might help you. If don't have installed nodejs take a look at the Build section to learn about it
 
  • Install the http-server  (-g installs globally)
    •   npm install http-server -g  
  • Once it is installed move to the dist/ folder of the theme (where index.html is located) and run
    •   http-server . -a 127.0.0.1 -p 8080
 
If everything goes fine you can now access to the app at http://127.0.0.1:8080/

Main folders explanation

 
dist/ folder
 
This folder contains the compiled files. This files are ready to deploy on your server.
 
master/ folder
 
This folder contains the source files. You will find the following folders inside
 
sass/ This folder contains the SCSS files for the core styles and bootstrap styles.
jsx/ Here you will find JSX files for React components. All this files are concatenated into the file app.bundle.js.
img/ Images used in the copy (only copied to dist folder)
server/ Contains json files used to emulate server calls
 

Custom code

JSX: Add a new component folder to create new ReactJS components and import them into the file App.jsx, then follow the code examples to associate components to routes.
 
SASS: create a custom folder inside the sass folder and @import your files from app.scss
 
 
 
The premise is, the less you change the downloaded code, the easier will be to apply any updates. Try always to keep your own code the most separated as possible from the package code to easily apply new updates when necessary.

 

 

Build

Node.js is a platform built on Chrome’s JavaScript runtime for easily building fast, scalable network applications.
 
Gulp is a task manager, you can define different tasks to run certain commands. Those commands does the compilation job for JSX and SASS files
 
Bower is a dependency manager, it works by fetching and installing packages from all over, taking care of hunting, finding, downloading, and saving the stuff you’re looking for. Bower keeps track of these packages in a manifest file, bower.json.
 
The package includes under the master/ folder the file gulpfile.js, package.json and bower.json to install the required components to compile the source files.
 
Installing tools
 
The following steps are intended to be an orientation guide, if you are not experienced with this you will need to learn more about it from Google :)
 
  • To install node and npm, go to http://nodejs.org/
  • Run npm install -g bower to install bower to manage dependencies
  • Download and install GIT for your platform http://git-scm.com/downloads
 
Once you have all tools installed
 
  • Open a terminal, in the root folder, run the command npm install. This command will install gulp and all project dependencies.
  • Then, to install vendor dependencies, run bower install
  • Finally run gulp to start the task manager (see below for more information)
If everything goes fine, you should see the messages in the terminal telling you that most the task are done ok. The task will watch for files to compile them automatically all files when change.
 

Main Gulp tasks

 
This command will run the default task without minify assets
$ gulp
This taks will start the app with a server in your favourite browser
$ gulp serve
This taks will start the app with a server in your favourite browser in production mode
$ gulp serve --prod
This taks will generate the asset minified (without watch)
$ gulp build --prod
 

Webpack and React Hot Loader

Internally, the application is built using WebPack which is a tool that analyze the code and builds a script ready for web browsers.

The configuration of webpack is included in files webpack.config.dev.js and webpack.config.prod.js for each environment respectively.

For development stage, gulp will enable a webpack plugin called react-hot-loader which allows to reload components easily without waiting to recompile the entire app.

 

 

Usage

Layout

Layout can be changed via the following classed applied to the body tag
 
.layout-fixed: Makes navbars become fixed while the user can scroll only content
 
.layout-boxed: Limits the width of the main wrapper element
 
.aside-collapsed: Condenses the sidebar showing only icons
 
.aside-collapsed-text: Condenses the sidebar showing icons and Text
 
.aside-toggle: used internally for mobiles to hide the sidebar off screen
 
.offsidebar-open: used internally to display the offsidebar component (formally the right sidebar)
 

JSX

The Javascript (JSX components) are bundled into one single file app.bundle.js that contains all react components declared under the master/jsx folder
 
The application start using the file App.jsx. This import all components and declare all routes associating components to each route.
 

Routes

 
To declare route we use react-router which is basically a routing library based on React components
You can note that according to the layout you are using is necessary to wrap each route with a Base* layout component. Example
 
ReactDOM.render(
    <Router history={browserHistory}>
        <Route path="/" component={Base}>

            {/* Default route*/}
            <IndexRoute component={DashboardV1} />

            ...

        </Route>

        {/*Horizontal Layout*/}
        <Route path="/" component={BaseHorizontal}>
            ...
        </Route>

        {/*Pages*/}
        <Route path="/" component={BasePage}>
            ...
        </Route>

        {/* Not found handler */}
        <Route path="*" component={NotFound}/>

    </Router>,
    document.getElementById('app')
);
 

Bootstrap components

 
To make the application the most ReactJS-friendly we have included react-bootstrap which is basically the popular framework rebuilt for React

Writing components

 
Each component is declared using the following basic code using ES6
import React from 'react';
import ContentWrapper from '../Layout/ContentWrapper';

class MyComponent extends React.Component {

    render() {
        return (
            <ContentWrapper>
                ...
            </ContentWrapper>
            );
    }

}

export default MyComponent;
 
ContentWrapper is a component that must be used to wrap each view in the app.
 
Then from the App.jsx you can import components with
import MyComponent from './path/to/MyComponent';
 
To keep the Javascript that controls the behavior of plugins separated from the associated component we use the following convention to name files (optional)
 
MyComponent.run.js
 
As you can see, we are just adding the "run" word and the extension is JS
Inside you can declare a function to be exported
 
export default function() {
    // code..
}
 
Then from the component JSX you can import this file
 
import MyComponenttRun from './MyComponentt.run';
 
And using the componentDidMount() call the imported function
 
componentDidMount() {
   MyComponenttRun();
}
 

SASS

SASS files are used to compile the main application styles into the file app.css and the custom Bootstrap styles into bootstrap.css
 
Also the app-rtl.css and bootstra-rtl.css are automatically generated with the same styles but inverted for RTL layout. 

Vendor

vendor.bundle.js: contains the vendor  scripts used in the application
vendor.bundle.css: contains the vendor styles used in the application
 
To edit the assets included in vendor bundles file, edit the paths in file vendor.json located under the root folder

Vendor Updates

To update vendor files via bower you can edit the bower.json file by adding the last version you want to download.
 

 

Themes

To change the color scheme:

From SASS files

 
Edit the SASS files in folder master/sass/app and the file master/sass/bootstrap/variables.scss to use the color you want.
Like Bootstrap, most of the colors are based on $brand-* variables.
 
You can also edit the files in master/sass/themes folder to create your own set of color schemes. This files must be included after the app.css in order to override the default color set.
 
Changing the theme colors from SASS files helps you to avoid bloating your css by not double declaring your color rules.
 

Default Theme

 
To set a default theem just include in file index.html the theme stylesheet of your choice right after the the main stylesheet (app.css)

Seed Project

This project is an application skeleton. You can use it to quickly bootstrap your ReactJS webapp projects and dev environment for these projects. 
The seed app doesn't do much and has most of the feature removed so you can add theme as per your needs just following the demo app examples.
 
This project is provided in order to start with the template using a different approach. Usually, templates will come with all features working and you need to remove them one by one in the way you don't need them.
 
With the seed project you can start adding custom features and others from the full project to make grow your app.
 
Since the files and structure is the same for the full and seed projects, you can save time using comparison tools that allows to apply changes from full features project into the seed project
